# IC Autorun — UX Review Round N (2026-07-19)

> Read-only code-level UX review conducted via 3 parallel `orchestrator-verifier`
> subagents (read-only — parallel-safe per AGENTS orchestration rules). Scope:
> all 21 `lib/screens/*.dart` + 53 `lib/widgets/*.dart` (~30k LOC). All three
> reports merged here; this file is the single source of truth for what was
> found and what Phase F will address.
>
> Methodology note: visual screenshots captured at `kDesktopDpr=1.0` (1440x900)
> proved too low-resolution for reliable vision-MCP OCR — many calls returned
> hallucinated text. Code-level review is the primary signal; vision was used
> only for layout sanity-checks on the Web bundle (Tier B Playwright captures
> at native Chromium DPR read correctly). A follow-up bump of `kDesktopDpr`
> to 2.0 is filed as Phase H tech-debt.

## Verdict

**NEEDS REVISION.** Three CRITICAL bugs and ~25 HIGH-severity issues found.
The code is broadly competently structured (typed state machines, empty-state
variants, lazy-loaded tabs, hover-reveal desktop actions, keyboard shortcuts
exist), but the **trust-signal layer is effectively absent**, there are
**real bugs on money and data-loss paths**, several **destructive-action
patterns are too thin** for a tool that downloads and executes third-party
code, and **dark-mode is broken across the key-management surface**.

## CRITICAL (data loss / money path / wrong action)

### CRIT-1 — Recovery-codes screen traps the user into lying
`lib/screens/recovery_codes_screen.dart:48`

- `automaticallyImplyLeading: false` AND no back/close button anywhere.
- Continue button (line 232) is disabled unless `_hasConfirmed == true`.
- The ONLY escape is to tick "I have saved these recovery codes in a secure
  location" (line 220) — even if the user hasn't.
- No Download `.txt` button — only `Copy All Codes` (line 190).
- Impact: trains users to tick the box without saving → permanent vault
  data loss. Single biggest data-loss risk in the identity surface.
- Fix: AppBar with back-arrow + warn-on-leave dialog ("Leave without saving?
  You'll need these codes to recover your vault.") + Download `.txt` button
  next to Copy.

### CRIT-2 — Wizard partial-failure leaves orphan duplicate-prone profiles
`lib/screens/unified_setup_wizard.dart:792-812`

- `createProfile` (line 792) persists the profile + keypair to secure
  storage **before** the network-bound `registerAccount` (line 800) runs.
- If registration fails (offline / 5xx / username race), the catch block
  (line 821) sets `_errorMessage = humanizeSecureStorageError(e)` — which
  misleadingly reads "Could not create the profile" even though it WAS
  created.
- On retry, `createProfile` runs AGAIN, producing a SECOND profile + keypair.
- Impact: silent orphan profiles + keypairs the user never intended; the
  wizard never reaches the success screen, so the first-run gate stays live.
- Fix: either roll back the persisted profile on registration failure, OR
  skip re-running `createProfile` on retry (treat as "profile exists, just
  retry registration") AND re-word the error to "Profile created locally,
  but marketplace registration failed: …".

### CRIT-3 — Currency label mismatch on script publish flow
`lib/widgets/quick_upload_dialog.dart:511`

- Upload form labels the price field `Price (ICP) *`.
- The rest of the app treats `script.price` as **USD**:
  - `lib/screens/scripts_screen.dart:760` passes it as `usdAmount` to ICPay.
  - `lib/screens/scripts_screen.dart:869` shows `Purchase required ($X.XX)`.
  - `lib/widgets/script_details_dialog.dart:362, 977, 1123` show `$X.XX`.
- An author who enters `5` thinking "5 ICP" silently lists their script at
  **$5 USD**.
- Impact: real money-path bug — wrong currency on a publish flow; under-
  or over-charges buyers by ~5-10x depending on ICP/USD.
- Fix: rename label to `Price (USD) *` with helper text `"USD, charged via
  ICPay. Set to 0 for free scripts."` (one-line change; no logic changes).

## HIGH — fixes that materially change a user's experience

### H-1 Trust signals absent on every script surface
`lib/widgets/scripts_list_item_tile.dart`, `lib/widgets/script_details_dialog.dart`,
`lib/widgets/script_execution_bottom_sheet.dart`

The entire product promise is **signed, sandboxed, downloadable + executable
scripts**. Today the user gets ZERO visible signal of either property at the
moment they decide to download or tap-run:

- No "Sandboxed" chip on tile subtitle or details header.
- `MarketplaceAuthor.isVerifiedDeveloper`, `authorPublicKey`, `uploadSignature`
  are checked internally but never surfaced to the user.
- Run-panel header (script_execution_bottom_sheet.dart:67-121) shows only
  title + expand + close — no sandboxed/identity/permission indicator while
  code executes.
- The marketplace tile omits downloads/rating at the decision point
  (marketplace_stats_banner returns `SizedBox.shrink()` on error with zero
  explanation — UX-13 from review B).

Fix: add a "Sandboxed ✓" + "Signed by {author}" chip pair to (a) tile
subtitle, (b) details dialog header next to author name, (c) run-panel
status row.

### H-2 Empty-library CTA inversion
`lib/widgets/scripts_empty_state.dart:110-118`

When the user lands on Scripts with empty library, the primary CTA is
"Create Script" and secondary is "Browse Marketplace". New users want to
FIND scripts, not author them. Invert.

### H-3 Dark-mode breakage across key management
25+ `Colors.white` hard-coded references in:
`add_account_key_sheet.dart` (6), `account_key_details_sheet.dart` (1),
`profile_menu.dart` (3), `quick_upload_dialog.dart` (2), `script_details_dialog.dart` (7),
`script_editor.dart` (1), `spotlight_overlay.dart` (1), `modern_empty_state.dart` (2),
`shimmer_loading.dart` (1), `animated_fab.dart` (1).

All render wrong colors in Dark theme. Affects Account, Dapps tour,
onboarding spotlight, marketplace details.

Fix: replace `Colors.white` with `theme.colorScheme.onPrimary` / `surface`
at each call site. The pattern is mechanical.

### H-4 Trust-grant dialog is all-or-nothing + missing principal warning
`lib/widgets/script_app_host.dart:611-633, 989-994`

- Only "Deny" + "Trust this dapp" (allow-always). No "Allow once" though
  `_showPermissionDialog` framework already supports the `allowLabel`
  parameter.
- Body says "any method, signed or anonymous" but never warns the dapp will
  **SEE YOUR PRINCIPAL** / can deanonymize.

Fix: restore the "Allow once" affordance + add a one-line warning: "The dapp
will see your principal and can identify you on every call."

### H-5 Unguarded destructive actions (4 paths)
- `lib/widgets/recent_calls_list.dart:73-78` — "Clear" wipes ALL call history
  with zero confirmation.
- `lib/widgets/bookmarks_list.dart:185-208` — Bookmark trash IconButton
  deletes immediately, no confirm, no undo.
- `lib/widgets/keypair_switcher_sheet.dart:174-179` — Tapping a profile tile
  instantly both selects AND pops; "Apply" button is dead UX.
- `lib/screens/import_keys_dialog.dart:53-55` — "Profile already exists.
  Delete it first" suggests destroying a profile (and its local-only data)
  to restore a backup. Should offer merge-or-replace or scope into a NEW
  profile (per AGENTS keypair-ownership rule).

Fix: add confirm dialogs (or 4s undo SnackBars) at each; rewrite import to
allow merge-or-replace.

### H-6 Wizard doesn't surface vault + passkey steps
`lib/screens/unified_setup_wizard.dart:798-812`

Per spec the wizard should offer vault setup + passkey enrollment as
optional final steps. Today both are buried behind the profile menu —
a first-time user with a marketplace account never sets up encryption or
passkey unless they explore unprompted. AccountRegistrationWizard (lines
573-594) DOES show a post-registration passkey prompt; unify the helper
and call it from both wizards.

### H-7 First-run wizard has no connectivity precheck
`lib/main.dart:382-410`, `lib/screens/unified_setup_wizard.dart:768-831`

User can reach the wizard while offline; createProfile succeeds locally,
registerAccount throws → CRIT-2 partial state. Add a connectivity probe at
wizard entry; if backend is unreachable, show an actionable panel BEFORE
letting the user type.

### H-8 "Canisters" tab is unexplained jargon
`lib/main.dart:585` (`kCanistersTabLabel = 'Canisters'`, defined at
`lib/screens/bookmarks_screen.dart:21`)

First-time users have no idea what "Canisters" means. Either rename or
attach a one-line info popover on long-press / hover.

### H-9 Raw exception strings in user-facing errors
- `lib/screens/vault_password_setup_screen.dart:175-178` — `"Failed to
  ${...} vault: $e"` leaks raw exception.
- `lib/screens/account_registration_wizard.dart:599` —
  `e.toString().replaceAll('Exception: ', '')` exposes raw text.
- `lib/screens/scripts_screen.dart:607, 748, 767, 891` — download failures
  show HTTP bodies / stack traces.

Fix: map known error types → friendly messages; keep raw detail behind a
collapsible "Show details" (pattern already exists in `_DappErrorView`).

### H-10 Fake progress bars
`lib/screens/scripts_screen.dart:522-530`, `lib/widgets/quick_upload_dialog.dart:237-246`

Download + upload progress is FAKED via `[0.3, 0.6, 0.9]` with
`Future.delayed(100ms)`. Misleads users about timing and erodes trust when
real timing differs. Drive from real byte/HTTP progress OR remove (use
indeterminate spinner).

### H-11 Three divergent "well-known canisters" catalogs
`lib/widgets/canister_call_builder.dart:36-46` vs `lib/widgets/well_known_canisters.dart:26-83`
vs `lib/services/canister_registry_service.dart` — three separate hard-coded
lists with DIFFERENT members. Call Builder omits ICLighthouse / Cyql / Kinic
/ Canistergeek that the Canisters tab shows.

Fix: replace with one shared const; AGENTS "single constant lives in ONE
place".

### H-12 No authenticated canister calls
`lib/widgets/canister_client_sheet.dart:321`

"Call" button only invokes `callAnonymous`; bridge supports
`callAuthenticated` but it's never used. Power users cannot sign canister
calls (transfers, votes, updates). Wire `callAuthenticated` when an active
keypair exists; show a "Signed as: <principal>" chip like the dapp runner.

## MEDIUM — convenience sample (24 more in raw reports)

- Wizard keyboard-incomplete: Enter doesn't submit on display-name /
  username / vault-password fields (`unified_setup_wizard.dart:375`,
  `account_registration_wizard.dart:155-202`).
- Vault password rules strict (12+ chars + 4 classes) with no strength
  meter, no generator (`vault_password_setup_screen.dart:72-92`).
- Vault unlock at 3 fails: passive suggestion, no inline "Use recovery code"
  CTA next to the message (`vault_unlock_screen.dart:110-112`).
- AppBar title "Get Started" duplicates primary CTA "Get Started"
  (`unified_setup_wizard.dart:316, 562`).
- "No account" pill on avatar (`profile_menu.dart:205, 1064`) — mildly
  shaming framing. Use "Local profile" or "Guest — tap to set up".
- Delete Profile warning omits that registered marketplace account becomes
  inaccessible (`profile_menu.dart:787-806`).
- `(see R-1)` milestone code leaks into user-visible error
  (`passkey_management_screen.dart:44`).
- Hard-coded `DD/MM/YYYY` date format — locale-ambiguous
  (`passkey_management_screen.dart:154-156`).
- `— see R-1` milestone code leak in passkey errors.
- Filters inconsistent: category persists to SharedPreferences;
  downloaded-only / favorites-only reset on navigation (`scripts_screen.dart:486-494`).
- Local script titles get `" (Marketplace)"` / `" (Copy)"` cruft appended
  (`scripts_screen.dart:540-543, 917, 1160`).
- Tile overflow menu author fallback "Unknown" — unfriendly
  (`scripts_list_item_tile.dart:131`).
- Tap-target 36×36 on hover-reveal actions — below Material's 48×48
  recommendation (`hover_reveal_actions.dart:161-166`).
- Three different relative-date formatters in the scripts surface.
- "Already Downloaded" menu item is a disabled dead weight inside a menu
  of actions (`script_context_menu.dart:184-189`).
- Upload "Tags (comma-separated)" plain text — error-prone entry for a
  structured list; marketplace already uses chips elsewhere.
- Generated snippet hardcodes a stub comment: `// Note: You'll need to set
  private_key_b64 or keypair_id` (`canister_call_builder.dart:195-200`).
- `_friendlyError` returns "Something went wrong" for any non-Format/
  Timeout exception — hides real canister error body
  (`canister_client_sheet.dart:171-187`). AGENTS mandates bodies surface.
- `_VecFieldController.toJsonValue` catches FormatException and falls back
  to raw text — silent type error sent to canister (`candid_smart_form.dart:570-576`).
- `_VariantFieldController.toJsonValue` always emits `{name: null}` even
  for variants with payload types — produces invalid Candid args
  (`candid_smart_form.dart:333-388`).
- "Clear All" filter button only appears when `activeFilters.length > 1`
  (`scripts_search_bar.dart:88-100`).
- Local-replica banner embeds shell commands as plain non-copyable text
  (`dapp_runner_screen.dart:99-100`).
- `?` shortcut uses `event.character == '?'` — layout-dependent; non-US
  keyboards cannot open shortcuts help (`keyboard_shortcuts.dart:119-133`).
- Well-known canister card Row overflows by 80px on narrow widths
  (`well_known_canisters.dart:144-145` — pre-existing, AGENTS-noted).
- Hard-coded `Colors.white` on key sheets (already in H-3).

## LOW — polish/consistency (~30 more in raw reports)

Notable: app brand mismatch `MaterialApp.title: 'ICP Autorun'` vs elsewhere
"IC Autorun" (`main.dart:270`); zombie widgets `InfoTooltip*` (4 variants,
none used in prod), `ContextualTip` (2 variants, unused); modern_empty_state
renders 3 decorative pulsing dots at the bottom (visual noise); "Recovery
hint" copy vague on vault_password_setup_screen.

## Top 7 highest-leverage Phase F fixes

1. **CRIT-1**: recovery-codes AppBar + Download .txt + warn-on-leave.
2. **CRIT-2**: rollback profile on registerAccount failure; reword error.
3. **CRIT-3**: rename `Price (ICP)` → `Price (USD)`.
4. **H-1**: add "Sandboxed ✓" + "Signed by {author}" chips to tile, details, run panel.
5. **H-2**: invert empty-library CTAs (Browse Marketplace primary).
6. **H-3**: replace 25+ `Colors.white` with theme tokens.
7. **H-4**: restore "Allow once" + add principal-visibility warning to trust dialog.

These 7 are scoped for Phase F. The remaining HIGH-severity items (H-5
through H-12) become follow-ups; MEDIUM/LOW items become backlog.
